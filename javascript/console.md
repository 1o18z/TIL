console.log 와 console.dir
=====
&nbsp;
### ☻ 차이점
&nbsp;
  
#### 1. 출력하는 자료형이 다르다.  
- dir 메소드는 문자열 표시 형식으로 콘솔에 출력한다.
   &nbsp;

#### 2. log는 값 자체에, dir은 객체의 속성에  

- log는 파라미터로 전달받은 값을 위주로 출력하는 반면, dir은 객체의 속성을 좀 더 자세하게 출력한다.

- dir을 출력해보면 객체의 유형이 먼저 출력되고, 특히 함수 부분에서는 클릭해서 펼쳤을 때 함수가 가진 속성들을 더 보여주는 모습을 확인할 수 있다.  (log는 펼쳐지지 않음)

&nbsp;
#### 3. log는 여러개, dir은 하나만  

- log는 여러 값을 쉼표로 구분해서 전달하면 전달받은 모든 값을 출력하지만, dir은 여러 값을 전달하더라도 첫번째 값만 출력한다.
&nbsp;
#### 4. DOM 객체를 다룰 때  

- 값에 좀 더 중점을 둔 log는 대상을 HTML 형태로 출력하고, 객체의 속성에 좀 더 중점을 둔 dir은 대상을 객체 형태로 출력한다.


&nbsp;
### 결론  
➣ 콘솔에서 값 자체를 확인하고 싶다면? -▻log메소드  
➣ 객체의 속성들을 살펴보고 싶다면? -▻ dir메소드
